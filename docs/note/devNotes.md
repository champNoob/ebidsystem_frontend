# 开发笔记

## 业务

### 初始路由定向

未登录访问时，默认重定向到 `/login`，而不是 `/register`

绝大多数系统假设：访问系统的人，更可能是已有账号的用户

策略：

| 场景         | 行为              |
| ------------ | ---------------- |
| 未登录访问任何业务路由  | 重定向 `/login`   |
| `/login` 页面     | 提供「去注册」入口   |
| `/register` 页面  | 提供「已有账号？去登录」 |
| 注册成功          | 自动跳转 `/login`（或自动登录） |
